# Checklist de Lanzamiento de Producto — Notion LATAM

**Aplica para**: Nuevos templates, bundles, cursos, membresía, o cualquier nuevo producto.  
**Responsable de completar**: Founder / Product Lead  
**Última actualización**: Abril 2026  

---

## FASE 1: PRE-LANZAMIENTO (2-3 semanas antes)

### Semana 3 antes del lanzamiento: Definición y desarrollo

**Producto**
- [ ] Definir el nombre final del producto y su posicionamiento en 1 línea
- [ ] Completar el desarrollo del template/curso/recurso
- [ ] Hacer prueba de usuario con 3 personas externas (beta testers)
- [ ] Documentar todos los cambios surgidos del beta testing
- [ ] Crear la guía de uso del producto (PDF o página en Notion)
- [ ] Verificar que el link de duplicado del template funcione correctamente
- [ ] Revisar el producto completo una última vez (QA final)

**Precio**
- [ ] Definir el precio en USD
- [ ] Confirmar que los métodos de pago están configurados
- [ ] Crear el producto en Stripe o Gumroad con el precio correcto
- [ ] Configurar el email automático post-compra con el link del producto
- [ ] Hacer una compra de prueba para verificar el flujo completo

---

### Semana 2 antes del lanzamiento: Contenido y marketing

**Página de ventas**
- [ ] Redactar la copy de la página de ventas (headline, beneficios, qué incluye, CTA)
- [ ] Crear screenshots o videos del producto para la página
- [ ] Publicar la página de ventas (puede ser en modo "oculto" o "próximamente")
- [ ] Revisar que todos los links de la página funcionan
- [ ] Verificar que la página se ve correctamente en móvil

**Contenido de lanzamiento (crear esta semana)**
- [ ] Artículo de blog relacionado con el problema que resuelve el producto
- [ ] 3 Reels o videos cortos mostrando el problema y la solución
- [ ] Carrusel para Instagram/LinkedIn explicando el valor del producto
- [ ] Email de anuncio para la lista (primer aviso, 1 semana antes)
- [ ] Story sequence de Instagram para el día del lanzamiento (mínimo 5 stories)

**Relaciones públicas**
- [ ] Identificar 5-10 cuentas en LATAM que puedan compartir el lanzamiento
- [ ] Contactar a partners potenciales con propuesta de colaboración
- [ ] Identificar grupos de Facebook/LinkedIn donde anunciar (con permiso)

---

### Semana 1 antes del lanzamiento: Preparación final

**Marketing**
- [ ] Programar todos los posts de redes sociales para la semana del lanzamiento
- [ ] Enviar email de "coming soon" a la lista con sneak peek
- [ ] Preparar las respuestas frecuentes anticipadas (FAQ)
- [ ] Confirmar disponibilidad de partners/colaboraciones

**Operaciones**
- [ ] Actualizar el catálogo de productos en la web y en el Notion interno
- [ ] Preparar el template de email post-compra
- [ ] Verificar que el equipo de soporte sabe todo sobre el nuevo producto
- [ ] Crear la página del producto en el CRM interno (para tracking de ventas)
- [ ] Preparar el script de WhatsApp para seguimiento post-lanzamiento

---

## FASE 2: LANZAMIENTO (Día D y semana de lanzamiento)

### Día del lanzamiento

**Mañana (antes de publicar)**
- [ ] Hacer compra de prueba final — verificar que todo el flujo funciona
- [ ] Confirmar que el servidor/plataforma está estable
- [ ] Tener el teléfono disponible para responder consultas

**Lanzamiento (hora acordada)**
- [ ] Publicar el primer Reel/video de lanzamiento en Instagram
- [ ] Publicar el post en LinkedIn
- [ ] Enviar el email de lanzamiento a toda la lista
- [ ] Publicar en Stories de Instagram con CTA y link en bio
- [ ] Actualizar el link en bio de Instagram
- [ ] Publicar en grupos de Facebook/WhatsApp donde está permitido
- [ ] Notificar a los partners/colaboradores para que compartan

**Durante el día del lanzamiento**
- [ ] Monitorear consultas en Instagram DM (responder en < 1 hora)
- [ ] Monitorear WhatsApp (responder en < 30 minutos)
- [ ] Monitorear email (responder en < 2 horas)
- [ ] Responder comentarios en los posts de redes
- [ ] Registrar cada venta en el CRM

**Noche del lanzamiento**
- [ ] Hacer conteo de ventas del primer día
- [ ] Publicar update en Stories ("X personas ya lo tienen")
- [ ] Responder todos los mensajes pendientes

---

### Días 2-7 post-lanzamiento

**Contenido de seguimiento**
- [ ] Día 2: Publicar un caso de uso o testimonial de beta tester
- [ ] Día 3: Email de seguimiento a quienes abrieron el email inicial pero no compraron
- [ ] Día 4: Reel con "detrás de escena" del producto
- [ ] Día 5: Carrusel con las preguntas más frecuentes recibidas
- [ ] Día 6: Recordatorio de urgencia (si hay precio de lanzamiento con límite de tiempo)
- [ ] Día 7: Email final si hay oferta de tiempo limitado ("últimas horas")

**Operaciones**
- [ ] Verificar que todos los compradores recibieron acceso al producto
- [ ] Responder todos los tickets de soporte dentro del SLA
- [ ] Documentar problemas surgidos durante el lanzamiento para mejorar el proceso

---

## FASE 3: POST-LANZAMIENTO (2-4 semanas después)

### Análisis del lanzamiento

**Métricas a documentar**
- [ ] Total de ventas del período de lanzamiento
- [ ] Ingreso total generado
- [ ] Tasa de conversión (visitas a la página / ventas)
- [ ] Canal con mayor conversión (email, Instagram, LinkedIn, WhatsApp)
- [ ] Feedback recibido de compradores (positivo y negativo)
- [ ] Problemas técnicos registrados durante el lanzamiento

**Documentación**
- [ ] Crear el registro del lanzamiento en Notion (datos + aprendizajes)
- [ ] Documentar qué funcionó bien para repetir en el próximo lanzamiento
- [ ] Documentar qué mejorar para el próximo lanzamiento
- [ ] Actualizar este checklist con lecciones aprendidas

### Optimización post-lanzamiento

- [ ] Recopilar testimoniales de los primeros compradores (usar Template WhatsApp #8)
- [ ] Publicar los primeros testimoniales en la página de ventas
- [ ] Ajustar la copy de la página de ventas según el feedback recibido
- [ ] Agregar una sección de FAQ a la página de ventas
- [ ] Crear contenido evergreen sobre el producto para SEO
- [ ] Configurar el producto en la secuencia de email onboarding para nuevos leads

### Integración al catálogo permanente

- [ ] Agregar el producto al catálogo oficial del sitio web
- [ ] Incluir en la comparativa de templates/servicios de la web
- [ ] Actualizar los bundles si aplica
- [ ] Entrenar al equipo de soporte sobre el producto con sesión de 30 minutos
- [ ] Crear el artículo de ayuda en el centro de ayuda sobre este producto
