## 🇺🇸 ENGLISH

### Overview

The **Product Roadmap Template** for Monday.com is the strategic command center for product and engineering teams that need to align business goals with development execution. From high-level roadmap planning to day-to-day sprint management, this template provides a seamless workflow that connects product vision to engineering delivery — and keeps everyone on the same page without endless meetings.

Built for modern product teams running agile methodologies, this template supports the full product development lifecycle: ideation → backlog grooming → sprint planning → active development → QA → release. Bug tracking is integrated directly into the workflow so that critical issues never fall through the cracks.

### What's Included

**3 Core Boards:**

1. **Features Board** — The master product backlog. Each feature/initiative is captured with: title, description, business value score, effort estimate (story points or t-shirt sizing), priority, target release quarter, status, and linked user stories. Features can be tagged by product area (Core, Growth, Monetization, Infrastructure) and stakeholder origin (Customer Request, Internal, Strategic).

   The board includes a **Now / Next / Later** view that gives leadership a clean quarterly roadmap view without exposing all the backlog detail.

2. **Sprints Board** — The active development execution board. Organized by sprint (2-week cycles by default), each sprint contains all the user stories and tasks committed for that sprint. Tracks: story points planned, story points completed, sprint velocity, assignee, status, and QA notes. Includes a **Burndown View** that automatically calculates remaining work vs. sprint timeline.

3. **Bugs Board** — A dedicated bug tracking board with severity levels (P0 Critical / P1 High / P2 Medium / P3 Low), status workflow (Reported → Triaged → In Progress → Fixed → Verified), affected version, reporter, assignee, and steps to reproduce. P0 and P1 bugs automatically trigger notifications to the engineering lead.

**Visual Product Roadmap** — A timeline view covering the current and next two quarters, organized by product area. Color-coded by priority and status. Can be exported as a shareable URL or PDF for stakeholder presentations and investor updates.

**Velocity Tracking Dashboard** — A rolling 12-sprint velocity chart showing completed story points per sprint, sprint-over-sprint trend, and team capacity utilization. Useful for sprint capacity planning and predictability improvement.

**Feature Portfolio View** — A grouped view of all features by product area, showing total story points, estimated quarters to complete, and business value vs. effort matrix to help prioritize the backlog.

### Automations (11 Total)

- Immediately notify engineering lead and PM via Slack when a P0 (Critical) bug is created
- Auto-create a sprint retrospective template when a sprint is marked as "Complete"
- Remind assignee when a sprint task has no update for 2 business days
- Auto-calculate sprint completion percentage as tasks are marked done
- Move features from "Now" to "Next" automatically when target quarter passes
- Alert PM when sprint velocity drops more than 20% from the rolling 4-sprint average
- Auto-link bug reports to the affected feature in the Features board
- Notify stakeholder when a feature they requested is moved to "In Development"
- Generate release notes draft when a sprint is completed
- Alert QA lead when all development tasks in a sprint are marked "Ready for QA"
- Send weekly product newsletter to stakeholder mailing list with feature status updates

- Product teams at SaaS companies running 2-week agile sprints
- CTO and VP of Product offices needing executive-level roadmap visibility
- Startup product teams moving from Jira/Linear to a more flexible, visual tool
- Product managers who need to communicate roadmap status to investors and board
- Engineering teams that want to integrate bug tracking with sprint planning

### Setup Time

Full setup: 45–60 minutes. Includes sprint cadence configuration guide, story point scale definition, and pre-built example sprint with 10 sample tickets to demonstrate automation flows.

## 🇪🇸 ESPAÑOL

### Descripción General

El **Template de Product Roadmap** para Monday.com es el centro de comando estratégico para equipos de producto e ingeniería que necesitan alinear los objetivos de negocio con la ejecución del desarrollo. Desde la planificación del roadmap de alto nivel hasta la gestión diaria de sprints, este template proporciona un flujo de trabajo sin interrupciones que conecta la visión del producto con la entrega de ingeniería — y mantiene a todos en sintonía sin reuniones interminables.

Construido para equipos de producto modernos que ejecutan metodologías ágiles, este template soporta el ciclo de vida completo del desarrollo de producto: ideación → refinamiento del backlog → planificación de sprint → desarrollo activo → QA → lanzamiento. El seguimiento de bugs está integrado directamente en el flujo de trabajo para que los problemas críticos nunca caigan en el olvido.

### Qué Incluye

**3 Tableros Principales:**

1. **Tablero de Features** — El backlog maestro del producto. Cada feature/iniciativa se captura con: título, descripción, puntuación de valor de negocio, estimación de esfuerzo (story points o tallas de camiseta), prioridad, trimestre de lanzamiento objetivo, estado y user stories vinculadas. Las features se pueden etiquetar por área de producto (Core, Crecimiento, Monetización, Infraestructura) y origen del stakeholder (Solicitud de Cliente, Interna, Estratégica).

   El tablero incluye una vista **Ahora / Después / Más tarde** que brinda a la dirección una vista limpia del roadmap trimestral sin exponer todos los detalles del backlog.

2. **Tablero de Sprints** — El tablero de ejecución del desarrollo activo. Organizado por sprint (ciclos de 2 semanas por defecto), cada sprint contiene todas las user stories y tareas comprometidas para ese sprint. Rastrea: story points planeados, story points completados, velocidad del sprint, asignado, estado y notas de QA. Incluye una **Vista de Burndown** que calcula automáticamente el trabajo restante vs. el cronograma del sprint.

3. **Tablero de Bugs** — Un tablero dedicado de seguimiento de bugs con niveles de severidad (P0 Crítico / P1 Alto / P2 Medio / P3 Bajo), flujo de trabajo de estado (Reportado → Clasificado → En Progreso → Corregido → Verificado), versión afectada, reportador, asignado y pasos para reproducir. Los bugs P0 y P1 automáticamente activan notificaciones al líder de ingeniería.

**Roadmap Visual de Producto** — Una vista de línea de tiempo que cubre el trimestre actual y los próximos dos trimestres, organizada por área de producto. Con código de color por prioridad y estado. Se puede exportar como URL compartible o PDF para presentaciones a stakeholders y actualizaciones a inversores.

**Dashboard de Seguimiento de Velocidad** — Un gráfico de velocidad de los últimos 12 sprints que muestra story points completados por sprint, tendencia sprint sobre sprint y utilización de la capacidad del equipo. Útil para la planificación de capacidad del sprint y la mejora de la previsibilidad.

**Vista de Portfolio de Features** — Una vista agrupada de todas las features por área de producto, mostrando story points totales, trimestres estimados para completar y una matriz de valor de negocio vs. esfuerzo para ayudar a priorizar el backlog.

### Automatizaciones (11 en total)

- Alerta al líder de QA cuando todas las tareas de desarrollo en un sprint se marcan "Listos para QA"
- Envío de newsletter semanal de producto a la lista de correo de stakeholders con actualizaciones de estado de features
